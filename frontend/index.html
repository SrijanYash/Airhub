Understanding the Air Quality Index (AQI)

The AQI is a scale used to report the quality of the air. It focuses on health effects you may experience within a few hours or days after breathing polluted air. Below is a breakdown of the AQI categories.

AQI Level Category Health Implications
1 Good Air quality is considered satisfactory, and air pollution poses little or no risk.
2 Fair Air quality is acceptable; however, some pollutants may be a moderate health concern for a very small number of people.
3 Moderate Members of sensitive groups may experience health effects. The general public is not likely to be affected.
4 Poor Everyone may begin to experience health effects; members of sensitive groups may experience more serious health effects.
5 Very Poor Health alert: everyone may experience more serious health effects. Everyone should avoid all outdoor exertion.

Abstract

Air pollution has emerged as one of the most critical global challenges, affecting human health, climate, and the environment. Traditional air quality forecasting models rely heavily on centralized data collection, which raises privacy concerns, increases communication costs, and limits scalability. This paper presents AirHub, a federated learning-based framework designed to predict air quality and weather conditions for specific geographical regions while preserving data privacy. The proposed system integrates distributed learning models that train locally using weather and pollutant data obtained from public APIs and sensor networks. Instead of sharing raw data, each client transmits model updates to a central aggregator, which refines a global predictive model. The aggregated model forecasts the Air Quality Index (AQI) and pollutant levels for upcoming time intervals. The backend, implemented in Python for model training and Java Spring Boot for orchestration, communicates with a web-based frontend to visualize real-time and predicted AQI levels. Experimental results demonstrate that federated learning provides comparable accuracy to centralized methods while enhancing privacy and scalability in environmental forecasting.

1. Introduction

Air pollution is a significant threat to environmental sustainability and public health. Accurate forecasting of air quality is crucial for developing effective mitigation strategies, urban planning, and public awareness. Existing centralized approaches collect large volumes of environmental data in one location for model training. While this improves prediction accuracy, it also leads to several challenges such as data privacy risks, communication overhead, and dependency on centralized infrastructure.

Recent advancements in Federated Learning (FL) have enabled decentralized model training where raw data remains local, and only model parameters are exchanged between clients and a central server. This paradigm offers a promising direction for privacy-preserving and scalable environmental modeling.

This research introduces AirHub, a web-based intelligent framework that integrates federated learning techniques for air quality forecasting. AirHub gathers weather data and pollutant metrics from APIs (e.g., OpenWeatherMap, AirNow) and uses deep learning algorithms to predict future air quality indicators such as PM2.5, PM10, CO₂, and the overall AQI score. The trained global model’s predictions are served via a user-friendly frontend interface, providing real-time and forecasted environmental insights to users.

2. Literature Review

Several studies have explored machine learning-based techniques for air quality forecasting and pollution monitoring. Traditional models such as ARIMA and Support Vector Regression (SVR) have been used to capture temporal dependencies in air quality data, but they struggle with high-dimensional, non-linear environmental datasets. Deep learning models like Long Short-Term Memory (LSTM) networks and Convolutional Neural Networks (CNNs) have achieved improved performance by modeling spatial-temporal relationships in pollutant trends.

However, most existing works employ centralized learning architectures, which aggregate all data in a single location for training. Li et al. (2021) demonstrated a CNN-based air quality prediction system that achieved high accuracy but required continuous centralized data collection from multiple stations. Similarly, Zhang et al. (2022) proposed an LSTM model trained on city-wide datasets, which suffered from data privacy and scalability issues.

Federated Learning (FL) has recently gained attention in environmental data processing. Kairouz et al. (2021) and Yang et al. (2022) highlighted how FL enables collaborative model training without data sharing. Few works have extended this concept to air quality prediction; Chen et al. (2023) implemented a decentralized FL-based model for AQI prediction, reporting minimal accuracy loss compared to centralized models.

3. System Architecture and Flow

The architecture of AirHub consists of three main layers — Data Source Layer, Federated Learning Layer, and Application Layer, connected through APIs and web services.

Workflow Summary:

  • The system fetches weather data from APIs.
  • Each local node trains its model.
  • Model weights are sent to the aggregator.
  • The aggregator updates and redistributes the global model.
  • Predictions are generated and displayed on the frontend dashboard.

4. Experimental Setup

4.1 Dataset and Data Collection

The dataset was obtained through public weather APIs such as OpenWeatherMap, AirNow, and IQAir, which provide hourly data for parameters including temperature, humidity, wind speed, PM2.5, PM10, CO, NO₂, and O₃ concentrations. The data was collected for multiple geographic regions to simulate a distributed sensor network.

4.2 Data Preprocessing

The data underwent handling of missing values using linear interpolation, min–max normalization, and conversion to time-series format (using sliding windows) for sequence prediction.

4.3 Model Configuration

Each local client trained a Long Short-Term Memory (LSTM) network. Key hyperparameters: epochs=50, batch_size=32, learning_rate=0.001, optimizer=Adam. Federated Averaging (FedAvg) was used for aggregation.

4.4 Deployment Environment

Python 3.10, TensorFlow Federated/Flower, Java Spring Boot, Flask REST interface, Docker containerization. Hardware: Intel i7, 16GB RAM.

5. Results and Analysis

The experimental evaluation focused on comparing the federated learning model (AirHub-FL) with a centralized baseline trained on combined data. The federated model achieved comparable accuracy (~96% of centralized performance) while reducing data communication and preserving privacy.

Model MAE RMSE R²

6. Conclusion and Future Work

This paper presented AirHub, a federated learning-based intelligent system for air quality forecasting and weather analysis. AirHub ensures data privacy by training models locally and aggregating only model parameters. The integration of LSTM, federated aggregation, and real-time web deployment provides a robust and scalable solution for smart environmental monitoring systems.

Key Outcomes:

  • Successful deployment across multiple simulated regions.
  • Comparable predictive performance to centralized models.
  • Seamless integration with Java Spring Boot and web visualization.

Future Work:

Integration with IoT sensors (MQTT/LoRaWAN), edge inference, transfer learning for faster adaptation, global deployment across cities, explainable AI modules for interpretability, and enhanced privacy with secure aggregation.
